How to qualify for better gigs

Premium jobs are ranked by trust, delivery history, profile quality and matching details.

1

Complete the profile

Add a clear 60+ character bio, headline, education, location, portfolio, rates, skills and payout details. Empty profiles are limited to starter matches.

2

Pass competency checks

Use the Starter test for low budget jobs, Standard 3000+ for KSh 3,000+ jobs and Premium 5000+ for KSh 5,000+ jobs. Each level is stricter.

3

Finish approved tasks

High-paying jobs rank better after several completed, approved tasks. Start with starter and standard gigs, then use reviews to move up.

4

Use the workspace well

Keep tasks updated, upload files to the vault, log time honestly, submit clean work and respond inside messages. Workspace history strengthens matching.

Quick target

Aim for a complete profile, the right competency tier, strong pitch, accurate skills, and at least 5 completed approved tasks before expecting the highest budget gigs to rank strongly.

Referral Rules

KSh 50 is paid only after the referred worker activates their account and completes one approved job.

  1. Share your referral code with a new user.
  2. The new user must enter your code while creating the account.
  3. The referred user must activate their worker account from the wallet.
  4. The referred user must complete one job and the employer must approve the work.
  5. Gigs Kenya pays KSh 50 once to your wallet. Self-referrals and duplicate phone accounts do not qualify.
You will receive notifications when someone joins with your code and when the bonus becomes payable.

Submit Work Guide

Workspace path

  1. Open My Gigs, then open the job Workspace.
  2. Use Overview for instructions and shortcuts.
  3. Use File Vault to add or edit Google Docs, Word, Drive, OneDrive or Dropbox deliverable links.
  4. Use Action Center to submit final work, approve work, request revision or handle extensions.
  5. Use Activity Log to confirm every major action is recorded.

Google Docs

  1. Open Google Docs and create the work document.
  2. Paste your final answer, attach images or tables if required, and name the file clearly.
  3. Click Share, change access to Anyone with the link can view, then copy the link.
  4. Open the workspace File Vault, add that link with a clear label, then submit in Action Center.

Word Online

  1. Create or upload the document in OneDrive or Word Online.
  2. Use Share, choose Anyone with the link can view, then copy the link.
  3. Add the Word link to the File Vault first. The final submission will be blocked until the vault has a file.

Google Drive / Microsoft Word file upload

  1. Upload your .docx, .xlsx, PDF, image or zip file to Google Drive, OneDrive or Dropbox.
  2. Open sharing settings and choose Anyone with the link can view.
  3. Copy the sharing link, add it in File Vault with a clear label like Final Work v1.
  4. Go to Action Center, add a short handover note, then Submit For Review.
Always test the link in a private browser window before submitting so the employer can open it. Do not send personal contacts in chat; use File Vault for links and Action Center for approvals.

Trust & Safety

Helpful safeguards for better marketplace behavior.

Gigs Kenya already includes practical product features that encourage better behavior: profile transparency, public reviews, competency checks, approval flows, activity logs, file history, dispute escalation, and notification records.

Safer collaboration habits

  • Use detailed job descriptions and written expectations.
  • Keep communication inside platform messages and workspace updates.
  • Review profiles, skill tags, ratings and recent reviews before hiring.
  • Use milestones, approvals and disputes when work becomes unclear.

Useful account tools

  • Notification history for important events
  • Activity logs inside workspaces
  • Review submissions after gig completion
  • Public reporting options on visible profiles
Terms

Clear working rules for better gigs.

Use Gigs Kenya for genuine, lawful work. Keep payments, approvals, workspace updates and important decisions inside the platform so both sides have a useful record.

For employers

  • Post accurate titles, categories, budgets, locations and deadlines.
  • Fund escrow before work begins and review submissions promptly.
  • Use written revision notes when changes are needed.
  • Release payment after accepted work is complete.

For workers

  • Apply only when you can meet the scope and deadline.
  • Keep deliverable links, notes and progress updates in the workspace.
  • Do not submit copied, unsafe, unlawful or misleading work.
  • Use the dispute flow when a job cannot be resolved in chat.

Payments and records

  • Top-ups require a valid 10-character M-Pesa code or paying phone number.
  • Pending deposits, withdrawals, escrow and applications appear in the ledger.
  • Workspace exports and invoice statements are user-generated records.
  • Admin review may be required for disputed or suspicious transactions.

Location and conduct

  • Location tags indicate where work happens or is coordinated.
  • Do not post scams, illegal services, harassment or private data requests.
  • Report profiles or jobs that look unsafe or misleading.
  • Account access, wallet actions and profile information should stay accurate.
